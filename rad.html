RAD

RAD (Spiral model) is a model used to pump out several prototypes. The model is split into four quadrants. Some merits are:

  • Ideal for risky and challenging operations where checkpoints are appreciated
  • Welcomes flexibility; meaning requirements can change
Some Disadvantages:
  • The management process of the Spiral model is advance, so expenisve
  • Code efficiancy is not prioritised
