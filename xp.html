XP

XP (Agile) is a model used to frequently write prototypes. This model involves two programmers that pair up to work together, whilst having daily meetings. Some merits are:

  • Quality of Code is very high
  • All work is written at a sustainable pace
  • Continuos testing
Some Disadvantages:
  • Two programmers needed in same location, so inconveniant
  • Customer is constantly involved in process which could not be an option
